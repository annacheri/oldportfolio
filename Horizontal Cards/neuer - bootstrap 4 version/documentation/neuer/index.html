Fonts / Typography

We have only used Google fonts in our demo, you can use any with in a minute.

Default is: Open Sans Condensed.

In index.html inside head replace our link to font with your desired one.

<link href='http://fonts.googleapis.com/css?family=Open+Sans+Condensed:300,300i,700' rel='stylesheet' type='text/css'>

Open style.css find font-family: property and change it.

body {
    font-family: 'Open Sans Condensed', sans-serif;
    font-size: 16px;
    /* Margin bottom by footer height */
    margin-bottom: 60px;
    background: #f4f4f4;
    -webkit-font-smoothing: antialiased;
    -moz-osx-font-smoothing: grayscale;
}

.cbp-popup-singlePage-counter {
    font: 400 14px/30px "Open Sans Condensed",sans-serif;
}


.cbp-l-project-details .btn {
    font: 400 12px/17px "Open Sans Condensed", sans-serif;
    padding: 8px 18px;
    border-radius: 0;
    background: #1f1f1f;
    transition: .3s;
}

Retina support

When device with retina display open a page, script checks each image on the page to see if there is a high-resolution version of that image on your server. If a high-resolution variant exists, the script will swap in that image in-place.

For example, if you have an image on your page like this "/img/my_image.png" script will check your server to see if an alternative image exists at this path "/img/my_image@2x.png"

Simplified, all you have to do is to include same image with double width and height and @2x added in file name, and in same folder as original image on your server. If you don’t include double sized image then script will leave original one.

For image that you want to check for double sized one everything is normal in HTML, e.g.:

<img src="contents/images/portfolios/600x600/1.jpg" alt="">

If you dont want script to search for retina image add data-no-retina attribute on img element. e.g.:

<img src="contents/images/portfolios/600x600/1.jpg" data-no-retina alt="">
